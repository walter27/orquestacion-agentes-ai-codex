# Coding Standards

## JavaScript

- Usar nombres descriptivos.
- Separar logica de datos y renderizado cuando sea posible.
- Exportar funciones testeables.
- Evitar funciones demasiado largas.
- No usar variables globales innecesarias.

## HTML

- El input debe tener label.
- Los botones deben tener texto claro.
- Usar atributos data-testid para pruebas.

## CSS

- Usar clases simples.
- No usar selectores fragiles.
- Mantener legibilidad antes que efectos visuales complejos.

## Prohibido

- Placeholders sin comportamiento real.
- Cambiar pruebas para ocultar errores.
- Marcar criterios como aprobados sin evidencia.
